## ИСР1.1 Применение основных команд Git на практике

### Инициализация репозитория  
#### `git init`

Для начала работы с системой контроля версий необходимо перейти в каталог проекта через терминал и выполнить команду инициализации. В результате Git начнёт отслеживать данный каталог, а внутри него появится служебная папка `.git`, содержащая всю информацию об истории изменений и конфигурации репозитория.

Команда `git init` предназначена для создания локального Git-репозитория.

### Проверка текущего состояния  
#### `git status`

Команда `git status` позволяет получить сведения о текущем состоянии проекта: есть ли изменённые файлы, добавлены ли они в индекс, а также присутствуют ли новые или неотслеживаемые элементы. При выполнении этой команды в только что созданном репозитории выводится информация об отсутствии изменений.

### Добавление файлов в индекс  
#### `git add`

Для того чтобы Git начал учитывать внесённые изменения, используется команда `git add`.  
Она может применяться как к отдельному файлу (`git add file.c`), так и к группе файлов по шаблону (`git add "*.c"`). Чтобы добавить в индекс все изменения в проекте, применяется команда `git add -A`.

### Фиксация изменений  
#### `git commit`

Коммит — это зафиксированное состояние проекта на определённый момент времени. По сути, это «снимок» репозитория, к которому можно вернуться в будущем.  
Перед выполнением коммита изменения должны быть добавлены в область подготовки. После этого выполняется команда `git commit`, сохраняющая текущее состояние проекта в истории.

### Отправка данных в удалённый репозиторий  
#### `git push`

Чтобы опубликовать локальные изменения в удалённом репозитории, используется команда `git push`. Она указывает, в какой удалённый репозиторий и в какую ветку необходимо передать коммиты.  
Например: `git push origin master`, где `origin` — имя удалённого репозитория, а `master` — целевая ветка.

### Получение копии репозитория  
#### `git clone`

Команда `git clone` позволяет скачать проект из удалённого репозитория и создать его полноценную локальную копию. При этом автоматически настраивается связь с удалённым источником (например, GitHub).

### Обновление локального репозитория  
#### `git pull`

Если в удалённом репозитории появились новые изменения, их можно загрузить в локальную версию проекта с помощью команды `git pull`.  
Пример: `git pull origin master`. Команда объединяет получение изменений и их слияние с текущей веткой.

### Работа с ветками  
#### `git branch`

По умолчанию основной веткой проекта является `master`. Для создания новой ветки используется команда `git branch <имя_ветки>`.  
После выполнения команды новая ветка будет точной копией основной.

При повторном вызове `git branch` отобразится список всех существующих веток, где активная помечается специальным символом.  
Чтобы переключиться на другую ветку, применяется команда `git checkout <имя_ветки>`.

### Объединение веток: пошаговый пример  

**Шаг 1.**  
В ветке с новой функциональностью создаётся файл `feature.txt`, после чего он добавляется в индекс и фиксируется коммитом.

**Шаг 2.**  
После завершения работы выполняется возврат в основную ветку `master`.

**Шаг 3.**  
Так как файл был создан в другой ветке, в основной версии проекта он отсутствует. Для переноса изменений используется команда `git merge`, объединяющая ветку с новой функциональностью с основной.

**Шаг 4.**  
После успешного слияния ветка с реализованной функцией становится ненужной и может быть удалена.
